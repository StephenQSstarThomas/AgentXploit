## Vulnerability Details

### 🟡 MEDIUM Severity (2 findings)

#### 1. **Voice-Call Webhook Signature Verification Bypass**

- **Type**: Insecure Configuration / Authentication Bypass
- **Severity**: MEDIUM
- **CWE**: CWE-16
- **Location**: `extensions/voice-call/src/config.ts:432-476`

**Description**: Configuration validator only enforces Telnyx public key when `skipSignatureVerification=false`, allowing deployments with webhook signature verification completely disabled.

**Attack Vector**:
- Set `skipSignatureVerification=true` in config
- Deploy without `telnyx.publicKey`
- Configuration validator passes
- Craft unsigned POST to `/voice/webhook`
- Spoof Telnyx events (`call.speech`, `call.ended`)
- Inject arbitrary transcripts
- Control call flows
- Execute malicious agent prompts

**Impact**: Unauthenticated webhook spoofing → arbitrary call control → AI prompt injection → account takeover

**Mitigation**: Remove skipSignatureVerification or require explicit dev mode + NODE_ENV check. Always enforce provider credentials.

---

#### 2. **Notify-Mode Caller ID Spoofing**

- **Type**: Missing Authentication
- **Severity**: MEDIUM  
- **CWE**: CWE-601
- **Location**: `extensions/voice-call/src/manager/outbound.ts:229-274`

**Description**: Inbound policy (`allowlist`/`open`) trusts caller ID without verification. Notify-mode auto-speaks messages and hangs up without challenge-response.

**Attack Vector**:
- Configure inbound policy with allowlist
- Spoof caller ID to match allowlisted number
- Auto-connect granted based on spoofed number
- Receive sensitive notify messages
- Repeat to drain credits or trigger MFA

**Impact**: Information disclosure via spoofed caller IDs, financial abuse, social engineering

**Mitigation**: Implement PIN/DTMF challenge, integrate STIR/SHAKEN attestation, add rate limiting

---

### 🟢 LOW Severity (1 finding)

#### 3. **World-Readable Call Logs**

- **Type**: Insecure File Permissions
- **Severity**: LOW
- **CWE**: CWE-732
- **Location**: `extensions/voice-call/src/manager/store.ts:6-94`

**Description**: Call logs stored at `~/.openclaw/voice-calls/calls.jsonl` with OS default permissions (typically 0644), readable by other local users.

**Attack Vector**:
- On multi-user system, unprivileged user accesses log file
- Reads all call transcripts, phone numbers, conversations
- Bypasses application-level access controls

**Impact**: Local information disclosure, PII leak, compliance violations

**Mitigation**: Create directory with 0700, files with 0600. Add permission validation.

## Recommendations

### Immediate Actions
1. Remove `skipSignatureVerification` option or gate behind dev-only mode
2. Implement caller verification (PIN/DTMF) for notify-mode
3. Set restrictive permissions (0700/0600) for call logs

### High Priority
1. Integrate STIR/SHAKEN caller attestation
2. Add per-caller rate limiting
3. Audit logging for auth failures
4. Startup permission validation
